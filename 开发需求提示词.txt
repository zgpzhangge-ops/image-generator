请基于 Minmax M2.1 模型的代码生成能力，开发一个可本地直接运行的图生图网页，核心通过调用 Silly Dream（wish.sillydream.top）的 API 对接 Nano banana 生图模型，实现上传底图 + 输入提示词生成新图核心功能，严格遵循前后端分离架构（保护 API 密钥），代码需原生、新手友好、注释清晰，且所有硬性要求无遗漏。
一、核心功能要求（需完整实现，无删减）
前端交互层：支持 PNG/JPG 图片上传 + 实时预览、生图提示词输入、重绘强度（0-1，步长 0.05）滑块实时显示数值、生成按钮加载态禁用、生成结果图片直接展示、精细化错误提示（未传图 / 未输提示词 / 后端请求失败 / API 调用错误均单独提示）；
后端转发层：作为中间层隔绝前端与第三方 API，核心保护 Silly Dream 的 API Key（绝对不出现 在前端代码中）、接收前端参数并校验、转发请求至 Silly Dream 图生图 API、解析返回结果并统一格式回传前端、全量捕获异常并返回清晰错误信息；
核心数据流转：前端自动将上传图片转为无前缀 Base64 编码 → 前端 POST 请求自有后端接口 → 后端携带 API Key 调用 Silly Dream API → 后端提取生成图（优先 URL，无则取 Base64） → 前端接收并展示图片（自动兼容 URL/Base64 格式）。
二、技术栈硬性要求（不可替换，新手友好）
前端：纯原生 HTML+CSS+JavaScript，无任何框架 / 第三方依赖，直接用浏览器打开即可运行，样式简洁实用（适配电脑端），代码无冗余；
后端：Python+Flask 框架（语法简单，适配新手），仅使用基础依赖（flask、requests、flask-cors），无复杂配置；
接口规范：后端仅提供 1 个 POST 接口 /api/gen_image，接收 3 个参数：image（底图无前缀 Base64）、prompt（生图提示词，字符串）、denoising（重绘强度，浮点型，默认 0.75）；
运行端口：后端默认运行在 localhost:3000，前端固定调用该地址，无需额外配置。
三、Silly Dream API 对接核心要求（精准匹配，不可出错）
请求头构造：必须携带 Authorization: Bearer 你的API Key 和 Content-Type: application/json；
请求体构造：严格按 Silly Dream 图生图 API 要求，必传参数：model（Nano banana 官方名称）、image（前端传的无前缀 Base64）、prompt（生图提示词）、denoising_strength（前端传的重绘强度），可选参数默认值：size: 512x512、n: 1（生成 1 张图）；
结果解析：优先提取 Silly Dream 返回结果中的图片URL，若无 URL 则提取b64_json字段，统一以image键返回给前端；
关键替换点：代码中用醒目注释 +### 需替换 ### 标注 3 处必须手动修改的信息，且仅这 3 处：①Silly Dream 的 API Key ②Silly Dream 图生图 API 正式请求地址 ③Nano banana 模型的官方准确名称。
四、代码交付与格式要求（适配 Trae 生成逻辑，易上手）
分文件提供：单独提供后端代码文件（app.py） 和前端代码文件（index.html），两文件独立无耦合，可分别放在不同文件夹；
极简运行步骤：每个文件上方附带3 步内的运行指引，包括依赖安装命令、后端启动命令、前端运行方式，无复杂操作；
注释规范：核心逻辑（Base64 转换、请求转发、参数解析）加行内注释，配置项 / 替换点加块注释，无需冗余注释，代码可读性优先；
异常捕获：①前端：校验必传参数（图片 / 提示词），捕获网络请求异常；②后端：捕获参数缺失、第三方 API 调用错误（401/403/422/429 等状态码）、平台返回无图片数据等所有异常，且所有异常均返回结构化 JSON（包含 code/msg/detail 字段），前端可直接解析展示；
跨域支持：后端开启全量跨域（允许所有源 / 请求方式），确保前端本地调用无跨域错误，无需额外配置。
五、前端交互细节要求（提升体验，必须实现）
图片上传后自动生成缩略预览图（200x200px，等比适配），Base64 转换时自动去除 data:image/png;base64, 前缀，无需用户手动处理；
重绘强度滑块拖动时，右侧数值实时同步更新，精准显示当前值；
点击 “生成” 按钮后，立即进入加载状态（显示 “正在生成，请稍候”），按钮禁用；生成完成 / 失败后，自动恢复按钮可用状态，隐藏加载态；
生成的图片按原尺寸（512x512）展示，无拉伸变形。
六、额外要求（调试友好，适配新手）
代码交付完成后，附带3 条核心调试排查指引，针对本地运行最常见的问题：①前端调不通后端 ②第三方 API 调用返回 401/422 错误 ③生成图片无结果 / 返回空，给出直接、可操作的解决方法。
